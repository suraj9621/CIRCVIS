Introduction

Welcome to the official CIRCVIS AI Waste Intelligence documentation. CIRCVIS is an enterprise-grade, context-aware computer vision platform designed to automate material identification and sorting in municipal waste facilities, recycling complexes, and smart-city hardware.

By using deep learning ensembles fine-tuned on real-world landfill datasets (such as RealWaste and TACO), CIRCVIS bypasses the common failure modes of generic vision algorithms—such as overlapping materials, severe occlusion, crushing, and dynamic lighting on conveyor belts.

💡 System Vision

CIRCVIS is built to power high-throughput sorting, providing a bridge between high-speed camera sensors and automated mechanical separators or edge robotics arms. It scales from heavy cloud infrastructure to low-power edge gateways.

Key Features

  • 89% Ensemble Accuracy: Soft-voting decision pipeline combining three high-performance deep architectures.
  • Ultra-Low Latency: 45ms inference via optimized MobileNetV2 edge weights.
  • Webcam & Video Feed Support: Real-time streaming capture and frame-by-frame conveyor sorting analysis.
  • ESG Sustainability Metrics: Integrated calculations for CO₂ emission offsets, decomposition logs, and material values.
  • Integrated Feedback Loop: Real-time operator override endpoint to retrain and continuously calibrate ML performance.

Quickstart Guide

Get the CIRCVIS FastAPI backend and web application running locally in under 3 minutes. The system requires Python 3.10+ and standard tools.

1. Clone and Prepare Environment

Create a Python virtual environment and install the required dependencies:

Shell 
# Create virtual environment
python -m venv venv

# Activate virtual environment (Windows)
venv\Scripts\activate

# Activate virtual environment (Linux / macOS)
source venv/bin/activate

# Install production dependencies
pip install -r requirements.txt

2. Run the Application

Use our universal startup script or direct python execution to fire up both the ML model services and static pages:

Shell (Direct Python) 
# Start the FastAPI server on port 8000
python -m uvicorn backend.app.main:app --reload
Windows Batch Script 
# Run using startup batch file
run.bat

The backend will immediately spin up. Open your browser and navigate to http://localhost:8000 to interact with the application dashboard and real-time classification demos.

Ensemble Models

Generic computer vision systems rely on a single model. If that model encounters an object at a strange angle, lighting distortion, or covered in grease, the prediction fails. CIRCVIS solves this using a **Soft-Voting Deep Learning Ensemble** that aggregates features from three distinct neural architectures.

Architecture Accuracy Latency (CPU) Model Size Primary Target Deployment
ResNet50 88.2% 120ms 97 MB Accuracy-critical cloud servers
MobileNetV2 84.1% 45ms 13 MB Edge robotics and IoT controllers
EfficientNetB0 87.4% 65ms 20 MB Balanced cloud/hybrid containers
Ensemble (All 3) 89.5% 200ms 130 MB Production conveyor belt sorting

The Soft-Voting Algorithm

During prediction, raw image tensors are loaded into all three classifiers simultaneously. Rather than selecting the majority class label (hard voting), our ensemble sums the output class probability vectors (softmax predictions) generated by each network:

P_final(c) = w1*P_resnet(c) + w2*P_mobilenet(c) + w3*P_effnet(c)

Weights are calibrated using a Dirichlet parameter optimizer based on validation performance, ensuring that models with superior specialized accuracy (such as ResNet's high organic precision) carry more influence in final output calculations.

POST /predict

POST /api/predict

Submits a single image file for real-time waste classification. The API expects standard `multipart/form-data` containing the image stream.

Request Payload

  • file: Binary file stream (supported: jpeg, png, webp. Maximum size: 10MB).

JavaScript Code Example

JavaScript 
const file = fileInput.files[0];
const formData = new FormData();
formData.append('file', file);

const response = await fetch('/api/predict', {
  method: 'POST',
  body: formData
});

const result = await response.json();
console.log(`Predicted Class: ${result.class_name} (${(result.confidence * 100).toFixed(1)}%)`);

POST /predict-batch

POST /api/predict-batch

Processes multiple image streams concurrently. Ideal for industrial pipelines using multi-camera sorting stations.

Request Payload

  • files: Array of binary file streams (maximum 30 items per request).

Python Integration Code Example

Python 
import requests

url = "http://localhost:8000/api/predict-batch"
files = [
    ('files', ('item1.jpg', open('waste1.jpg', 'rb'), 'image/jpeg')),
    ('files', ('item2.jpg', open('waste2.jpg', 'rb'), 'image/jpeg'))
]

response = requests.post(url, files=files)
data = response.json()

for item in data['predictions']:
    print(f"File: {item['filename']} -> Class: {item['class_name']}")

POST /feedback

POST /api/feedback

Enables operators to override incorrect AI classifications. The payload is written immediately to the calibration database, feeding the active training queue to fine-tune the ensemble core over time.

Request Schema (JSON)

JSON Schema

}

Edge Robotics Integration

For low-power waste complexes, executing deep ensembles in the cloud introduces latency and network stability issues. CIRCVIS supports direct edge deployment via optimized TensorRT models.

Raspberry Pi & Jetson Nano Optimization

We convert the lightweight MobileNetV2 architecture into quantized INT8 ONNX / TensorLite configurations. This slashes the memory footprint to 13 MB while maintaining a sub-50ms inference latency directly on hardware:

  • Jetson Nano Setup: Utilizes CUDA cores and TensorRT for real-time video stream evaluation.
  • Raspberry Pi Setup: Runs the quantization-aware TF Lite runtime utilizing 4 parallel ARM CPU threads.

⚙ Edge Robotic Arm Trigger

In edge implementations, classification callbacks send standard serial signals (e.g., Modbus/TCP or simple UART signals) to direct robotic arms, blowing pneumatic sorting nozzles or mechanical gates to correctly divert items at the exact millisecond.

Troubleshooting FAQ

Q: Why am I seeing degraded/mock model predictions instead of the real deep ensemble?

A: If the model weights (`.keras` files) are missing from the `models/` directory during backend startup, the FastAPI server boots in a safe diagnostic fallback mode using simulated predictions. To solve this, run our training pipeline to generate the production weights:

Shell 
python data/train_models.py --epochs 30

Q: My camera feed is failing to load on the demo interface. How do I fix this?

A: Ensure that the page is served over a secure connection (`https://` or `localhost`). Modern browsers block webcam access (getUserMedia) on unsecured HTTP origins. Also, verify that no other software is utilizing the camera resource.

Q: How do we resolve confusion between Paper and Metal classes?

A: Metallic cardboards or foil-lined boxes are a common edge-case challenge. You can tweak the classification threshold limits in `models/inference_calibration.json` by adding a small logit bias to Paper, or submit operator overrides using the `/api/feedback` interface to calibrate weights during the next retraining epoch.
